Custom Project
Dataset
For this project, my Kafka producer streamed data from the file data/sales.csv.
This dataset contains individual product sales transactions, where each row represents a single order.

Each record includes fields such as:

order_id

product_id

customer_id

region_id

unit_price

quantity

datetime

payment_method

device_type

referral_source

is_online

is_new_customer

I used the original sales dataset provided in the repository and did not modify the source CSV.

Kafka Messages
The Kafka producer reads each row from sales.csv and sends it as a JSON‑like dictionary into the Kafka topic product-sales-case.

Details:

Producer sends: one sales record at a time

Kafka topic: product-sales-case

Message key: the producer uses the region_id as the Kafka key

Message fields: I did not change the fields sent by the producer; they match the original dataset

Each message includes metadata added by Kafka, such as:

_kafka_key

_kafka_partition

_kafka_offset

Consumer Processing
My custom consumer (kafka_consumer_hildebrand.py) receives each message from Kafka and performs several steps:

Receives the raw message from the topic

Logs the message with a timestamp (my Phase 4 modification)

Validates required fields

Enriches the message using lookup tables:

regions.csv (tax rates)

products.csv

currencies.csv

discount_codes.csv

Calculates subtotal, tax, and total

Updates running statistics (min, max, average, total sales)

Writes valid records to:

data/output/consumed_sales.csv

data/output/sales.duckdb

Updates the live chart with each new sale

Counts consumed vs. skipped messages

My consumer processed 6 messages during the run shown in my logs.

Experiments
I completed two custom modifications:

Phase 4 Modification — Add Timestamps
I added a timestamp to every raw message log entry by inserting:

python
timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
LOG.info(f"[{timestamp}] {row}")
This made the logs clearer and easier to follow.
